# Contributing

Thanks for your interest in contributing!

Guidelines:
- Fork the repository and create a branch named `feature/xxx` or `fix/xxx`.
- Run linters and tests before submitting a PR.
- Write clear PR titles and descriptions; reference related issues when applicable.
- Do not commit secrets or credentials; use environment variables.

Workflow:
1. Fork → create branch → implement → push → open PR → review → merge.
2. For urgent security fixes, follow the security policy if present.

Contact:
- For questions/issues, open an Issue on the repository.
